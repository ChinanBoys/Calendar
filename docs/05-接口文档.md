| 项目 | 内容 |
| --- | --- |
| 前端 | Vue 3（Axios 调用） |
| 后端 | Java + Spring Boot（RESTful） |

## 0. 通用说明

### 0.1 统一响应格式

| 参数名 | 类型 | 是否必须 | 备注 |

| code | number | 必须 | 响应码，1 代表成功，0 代表失败 |
| msg | string | 非必须 | 提示信息 |
| data | object / object[] | 非必须 | 返回的数据 |

```json
{ "code": 1, "msg": "success", "data": null }
```

## 1. 语音解析模块

### 1.1 语音 / 文本解析
> 对应功能：**F-VOICE-01 语音转写 / F-VOICE-02 文本兜底 / F-VOICE-03 解析结果确认卡**（P1 首页中央语音按钮、⌨ 输入）。
> 操作数据库：写入 `voice_log`（仅落解析日志，**不创建事件**，保证「先确认后保存」）。

#### 1.1.1 基本信息
请求路径：/api/voice/parse
请求方式：POST
接口描述：前端把语音转写后的自然语言文本发给后端，后端调用大模型解析为结构化 JSON，返回给前端渲染确认卡。

#### 1.1.2 请求参数
格式：application/json
参数说明：

| 参数名 | 类型 | 是否必须 | 备注 |
| --- | --- | --- | --- |
| text | string | 必须 | 用户语音转写/输入的自然语言原文 |
| now | string | 非必须 | 客户端当前时间（ISO8601），缺省用服务端时间 |
| tz | string | 非必须 | 时区，缺省取用户设置 timezone |

请求参数样例：
```json
{
  "text": "我明天下午三点到五点有个会议，地点在3号会议室，提前15分钟提醒",
  "now": "2026-05-31T09:56:00+08:00",
  "tz": "Asia/Shanghai"
}
```

#### 1.1.3 响应数据
参数格式：application/json
参数说明：

| 名称 | 类型 | 是否必须 | 备注 |
| --- | --- | --- | --- |
| code | number | 必须 | 响应码，1 成功，0 失败 |
| msg | string | 非必须 | 提示信息 |
| data | object | 必须 | 解析结果 |
| &#124;- intent | string | 必须 | 意图：create/query/update/delete/unknown |
| &#124;- title | string | 非必须 | 标题（intent=create 时） |
| &#124;- startTime | string | 非必须 | 开始时间（已换算为绝对时间） |
| &#124;- endTime | string | 非必须 | 结束时间 |
| &#124;- allDay | boolean | 非必须 | 是否全天 |
| &#124;- location | string | 非必须 | 地点（null 表示未提及，前端高亮提示核对） |
| &#124;- reminderOffsets | number[] | 非必须 | 提醒提前分钟数列表 |
| &#124;- recurrence | string | 非必须 | 重复规则 none/daily/weekly/monthly |
| &#124;- queryStart | string | 非必须 | 查询起始时间（intent=query 时） |
| &#124;- queryEnd | string | 非必须 | 查询结束时间（intent=query 时） |
| &#124;- matchKeyword | string | 非必须 | 定位事件的关键词（intent=update/delete 时） |
| &#124;- confidence | object | 非必须 | 各字段置信度（0~1），前端据此高亮低置信度字段 |
| &#124;- rawText | string | 非必须 | 识别原文 |

响应数据样例（intent = create）：
```json
{
  "code": 1,
  "msg": "success",
  "data": {
    "intent": "create",
    "title": "会议",
    "startTime": "2026-06-01T15:00:00",
    "endTime": "2026-06-01T17:00:00",
    "allDay": false,
    "location": null,
    "reminderOffsets": [15],
    "recurrence": "none",
    "confidence": { "intent": 0.99, "title": 0.9, "startTime": 0.95, "location": 0.4 },
    "rawText": "我明天下午三点到五点有个会议，地点在3号会议室，提前15分钟提醒"
  }
}
```

响应数据样例（intent = query，「我明天有什么安排」）：
```json
{
  "code": 1,
  "msg": "success",
  "data": {
    "intent": "query",
    "queryStart": "2026-06-01T00:00:00",
    "queryEnd": "2026-06-01T23:59:59",
    "confidence": { "intent": 0.99 },
    "rawText": "我明天有什么安排"
  }
}
```

## 2. 事件管理模块

> 本模块响应中的事件对象（event）字段，**与 `event` 表字段一一对应**（snake_case → camelCase）。

事件对象（event）字段说明（多处复用）：

| 名称 | 类型 | 是否必须 | 对应数据库字段 | 备注 |
| --- | --- | --- | --- | --- |
| id | number | 必须 | event.id | 事件 ID |
| userId | number | 必须 | event.user_id | 所属用户 |
| title | string | 必须 | event.title | 标题 |
| startTime | string | 必须 | event.start_time | 开始时间 |
| endTime | string | 必须 | event.end_time | 结束时间 |
| allDay | boolean | 必须 | event.all_day | 是否全天 |
| location | string | 非必须 | event.location | 地点 |
| note | string | 非必须 | event.note | 备注 |
| recurrence | string | 必须 | event.recurrence | 重复规则 |
| status | number | 必须 | event.status | 1 有效 0 已删除 |
| createTime | string | 必须 | event.create_time | 创建时间 |
| updateTime | string | 必须 | event.update_time | 更新时间 |

### 2.1 创建事件
> 对应功能：**F-ADD-01 添加事件 / F-ADD-02 重复 / F-ADD-03 全天**（P2 确认卡【确认保存】）。
> 操作数据库：向 `event` 插入一行；按 `reminderOffsets` 向 `reminder` 插入多行；插入前执行冲突检测（F-CONFLICT-01）。

#### 2.1.1 基本信息
请求路径：/api/events
请求方式：POST
接口描述：保存确认卡中（可能被用户修正过的）结构化事件并落库。

#### 2.1.2 请求参数
格式：application/json
参数说明：

| 参数名 | 类型 | 是否必须 | 备注 |
| --- | --- | --- | --- |
| title | string | 必须 | 标题 |
| startTime | string | 必须 | 开始时间（ISO8601） |
| endTime | string | 必须 | 结束时间（ISO8601） |
| allDay | boolean | 非必须 | 是否全天，默认 false |
| location | string | 非必须 | 地点 |
| note | string | 非必须 | 备注 |
| recurrence | string | 非必须 | 重复规则，默认 none |
| reminderOffsets | number[] | 非必须 | 提醒提前分钟数列表，默认取用户 default_reminder_minutes |
| force | boolean | 非必须 | 是否忽略冲突强制保存（确认卡「仍然保存」时为 true），默认 false |

请求参数样例：
```json
{
  "title": "会议",
  "startTime": "2026-06-01T15:00:00",
  "endTime": "2026-06-01T17:00:00",
  "allDay": false,
  "location": "3号会议室",
  "recurrence": "none",
  "reminderOffsets": [15],
  "force": false
}
```

#### 2.1.3 响应数据
参数格式：application/json
参数说明：

| 名称 | 类型 | 是否必须 | 备注 |
| --- | --- | --- | --- |
| code | number | 必须 | 响应码，1 成功，0 失败 |
| msg | string | 非必须 | 提示信息 |
| data | object | 必须 | 返回的数据 |
| &#124;- created | boolean | 必须 | 是否已创建（检测到冲突且 force=false 时为 false） |
| &#124;- event | object | 非必须 | 创建成功时返回的事件对象（字段见模块开头，来自 event 表） |
| &#124;- conflicts | object[] | 非必须 | 冲突的事件列表（created=false 时返回） |
| &#124;&#124;- id | number | 非必须 | 冲突事件 ID（event.id） |
| &#124;&#124;- title | string | 非必须 | 冲突事件标题（event.title） |
| &#124;&#124;- startTime | string | 非必须 | 冲突事件开始时间（event.start_time） |
| &#124;&#124;- endTime | string | 非必须 | 冲突事件结束时间（event.end_time） |

响应数据样例（创建成功）：
```json
{
  "code": 1,
  "msg": "success",
  "data": {
    "created": true,
    "event": {
      "id": 123,
      "userId": 1,
      "title": "会议",
      "startTime": "2026-06-01T15:00:00",
      "endTime": "2026-06-01T17:00:00",
      "allDay": false,
      "location": "3号会议室",
      "note": null,
      "recurrence": "none",
      "status": 1,
      "createTime": "2026-05-31T09:56:30",
      "updateTime": "2026-05-31T09:56:30"
    }
  }
}
```

响应数据样例（检测到冲突，未创建）：
```json
{
  "code": 1,
  "msg": "时间冲突",
  "data": {
    "created": false,
    "conflicts": [
      { "id": 77, "title": "周会", "startTime": "2026-06-01T14:30:00", "endTime": "2026-06-01T15:30:00" }
    ]
  }
}
```

### 2.2 事件列表查询（时间范围 / 关键词 / 分页）
> 对应功能：**F-VIEW-01 今日 / F-VIEW-02 周月 / F-VIEW-03 语音查询 / F-VIEW-05 搜索**（P1 时间轴、P3 日历、P6 搜索）。
> 操作数据库：`SELECT` `event`（`user_id` + 时间范围 / 关键词，`status=1`）。

#### 2.2.1 基本信息
请求路径：/api/events
请求方式：GET
接口描述：按时间范围、关键词条件分页查询当前用户的有效事件。

#### 2.2.2 请求参数
参数格式：queryString
参数说明：

| 参数名称 | 是否必须 | 示例 | 备注 |
| --- | --- | --- | --- |

| to | 否 | 2026-06-01T23:59:59 | 结束时间 |
| keyword | 否 | 会议 | 关键词，匹配 title / location（搜索场景） |
| page | 是 | 1 | 页码，默认为 1 |
| pageSize | 是 | 10 | 每页记录数，默认为 10 |

请求数据样例：
```
/api/events?from=2026-06-01T00:00:00&to=2026-06-01T23:59:59&page=1&pageSize=10
```

## 附录 A：典型业务链路（前端 Vue → 接口 → MySQL）

以「我明天下午三点到五点有个会议」为例：

1. 用户在 P1 按语音按钮 → 前端 ASR 转文字 →
2. `POST /api/voice/parse`（写 `voice_log`）→ 返回 intent=create 的结构化字段 →
3. 前端渲染 P2 确认卡；可选 `GET /api/events/conflicts` 实时查冲突（读 `event`）→
4. 用户点【确认保存】→ `POST /api/events`（写 `event` + `reminder`）→
5. 返回创建的事件对象 → 前端刷新 P1 今日视图（`GET /api/events`）。

以「把明天的会改到四点」为例：
1. `POST /api/voice/parse` → intent=update, matchKeyword=会, patch.startTime →
2. 前端用关键词命中事件（`GET /api/events?keyword=会`）→ 确认卡 →
3. `PUT /api/events/{id}`（更新 `event`，重算 `reminder.fire_time`）。
