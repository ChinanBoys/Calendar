# 语音日历 VoiceCal —— 数据库设计文档

| 项目 | 内容 |
| --- | --- |
| 数据库 | MySQL 8.0 |
| 字符集 | utf8mb4 / utf8mb4_general_ci |
| 后端 | Java + Spring Boot（MyBatis-Plus / JPA） |
| 前端 | Vue 3 |
| 时间存储约定 | 统一使用 `DATETIME`，按业务时区写入；接口出参为 ISO8601 字符串 |
| 配套 | `01-需求文档(PRD).md`、`05-接口文档.md` |

> 命名规范：表名、字段名一律 snake_case；主键统一 `id BIGINT UNSIGNED AUTO_INCREMENT`；
> 每张业务表都有 `create_time` / `update_time` 审计字段；删除采用**逻辑删除**（`status` / `deleted`），以支持「删除可撤销」(F-DEL-02)。

---

## 1. 数据库 E-R 关系

```
            ┌──────────────┐
            │     user     │  用户 / 设置
            │  (用户+偏好)  │
            └──────┬───────┘
                   │ 1
        ┌──────────┼───────────────┬──────────────┐
        │ N        │ N             │ N            │ N
   ┌────▼─────┐ ┌──▼────────┐ ┌────▼──────┐  (语音日志)
   │  event   │ │ reminder  │ │ voice_log │
   │ (事件)   │ │ (提醒)    │ │ (解析日志)│
   └────┬─────┘ └───────────┘ └───────────┘
        │ 1
        │ N
   ┌────▼─────┐
   │ reminder │  一个事件可有多条提醒
   └──────────┘
```

- `user` 1 : N `event`：一个用户有多个事件。
- `event` 1 : N `reminder`：一个事件可设置多条提醒（如提前 1 天 + 提前 15 分钟）。
- `user` 1 : N `voice_log`：记录每次语音/文本解析，用于纠错与隐私清除。

## 2. 功能 → 数据库表 映射总览

> 回答「我用了某个功能，操作的是哪张表」。功能编号定义见 `01-需求文档(PRD).md`，接口见 `05-接口文档.md`。

| 功能编号 | 功能 | 主要操作表 | 操作类型 |
| --- | --- | --- | --- |
| F-VOICE-01/02/03 | 语音/文本解析 | `voice_log` | INSERT（落解析日志，不落事件） |
| F-ADD-01 | 语音添加事件 | `event` + `reminder` | INSERT |
| F-ADD-02 | 重复事件 | `event`（recurrence 字段） | INSERT |
| F-ADD-03 | 全天事件 | `event`（all_day 字段） | INSERT |
| F-VIEW-01 | 今日视图 | `event` | SELECT（按 user_id + 当天范围） |
| F-VIEW-02 | 周/月视图 | `event` | SELECT（按 user_id + 时间范围） |
| F-VIEW-03 | 语音查询日程 | `event` | SELECT（按解析出的时间范围） |
| F-VIEW-04 | 事件详情 | `event` + `reminder` | SELECT（按 id） |
| F-VIEW-05 | 搜索 | `event` | SELECT（关键词 LIKE + 分页） |
| F-EDIT-01/02 | 修改事件 | `event` + `reminder` | UPDATE（含重算 fire_time） |
| F-DEL-01/02 | 删除/撤销 | `event` | UPDATE（status 置 0 / 还原 1，逻辑删除） |
| F-CONFLICT-01 | 时间冲突检测 | `event` | SELECT（时间重叠查询） |
| F-REMIND-01 | 到点提醒 | `reminder` | SELECT/UPDATE（扫描 fire_time，置 sent） |
| F-REMIND-02 | 自定义提前量 | `reminder` | INSERT/UPDATE |
| F-SET-01 | 提醒/时区设置 | `user` | UPDATE |
| F-SET-02 | 隐私/语音数据 | `user` + `voice_log` | UPDATE / DELETE |
| F-ACCOUNT-01 | 登录/同步 | `user` | SELECT/INSERT |

## 3. 表结构定义

### 3.1 user（用户表 / 含个人偏好设置）
对应功能：F-ACCOUNT-01 账户、F-SET-01 偏好、F-SET-02 隐私。

| 字段 | 类型 | 是否必填 | 默认值 | 说明 |
| --- | --- | --- | --- | --- |
| id | BIGINT UNSIGNED | 是 | AUTO_INCREMENT | 主键，用户 ID |
| username | VARCHAR(50) | 是 | — | 登录名，唯一 |
| password | VARCHAR(255) | 是 | — | 密码（加密存储，BCrypt） |
| nickname | VARCHAR(50) | 否 | NULL | 昵称 |
| timezone | VARCHAR(64) | 是 | 'Asia/Shanghai' | 时区，用于相对时间换算（F-SET-01） |
| default_reminder_minutes | INT | 是 | 15 | 默认提醒提前分钟数（F-SET-01） |
| notify_enabled | TINYINT(1) | 是 | 1 | 通知总开关，1 开 0 关（F-SET-01） |
| voice_retention | VARCHAR(10) | 是 | '7d' | 语音原文保留策略：none/7d/forever（F-SET-02） |
| create_time | DATETIME | 是 | CURRENT_TIMESTAMP | 创建时间 |
| update_time | DATETIME | 是 | CURRENT_TIMESTAMP | 更新时间 |

```sql
CREATE TABLE `user` (
  `id`                       BIGINT UNSIGNED NOT NULL AUTO_INCREMENT COMMENT '用户ID',
  `username`                 VARCHAR(50)  NOT NULL COMMENT '登录名',
  `password`                 VARCHAR(255) NOT NULL COMMENT '密码(BCrypt加密)',
  `nickname`                 VARCHAR(50)  DEFAULT NULL COMMENT '昵称',
  `timezone`                 VARCHAR(64)  NOT NULL DEFAULT 'Asia/Shanghai' COMMENT '时区',
  `default_reminder_minutes` INT          NOT NULL DEFAULT 15 COMMENT '默认提醒提前分钟数',
  `notify_enabled`           TINYINT(1)   NOT NULL DEFAULT 1 COMMENT '通知开关 1开0关',
  `voice_retention`          VARCHAR(10)  NOT NULL DEFAULT '7d' COMMENT '语音保留策略 none/7d/forever',
  `create_time`              DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  `update_time`              DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  PRIMARY KEY (`id`),
  UNIQUE KEY `uk_username` (`username`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='用户表(含个人偏好设置)';
```

### 3.2 event（事件表）
对应功能：F-ADD-*、F-VIEW-*、F-EDIT-*、F-DEL-*、F-CONFLICT-01。这是核心表。

| 字段 | 类型 | 是否必填 | 默认值 | 说明 |
| --- | --- | --- | --- | --- |
| id | BIGINT UNSIGNED | 是 | AUTO_INCREMENT | 主键，事件 ID |
| user_id | BIGINT UNSIGNED | 是 | — | 所属用户（FK → user.id） |
| title | VARCHAR(100) | 是 | — | 事件标题 |
| start_time | DATETIME | 是 | — | 开始时间 |
| end_time | DATETIME | 是 | — | 结束时间 |
| all_day | TINYINT(1) | 是 | 0 | 是否全天，1 是 0 否（F-ADD-03） |
| location | VARCHAR(200) | 否 | NULL | 地点 |
| note | VARCHAR(500) | 否 | NULL | 备注 |
| recurrence | VARCHAR(20) | 是 | 'none' | 重复规则：none/daily/weekly/monthly（F-ADD-02） |
| status | TINYINT(1) | 是 | 1 | 状态：1 有效 0 已删除（逻辑删除，支持撤销 F-DEL-02） |
| create_time | DATETIME | 是 | CURRENT_TIMESTAMP | 创建时间 |
| update_time | DATETIME | 是 | CURRENT_TIMESTAMP | 更新时间 |

```sql
CREATE TABLE `event` (
  `id`          BIGINT UNSIGNED NOT NULL AUTO_INCREMENT COMMENT '事件ID',
  `user_id`     BIGINT UNSIGNED NOT NULL COMMENT '所属用户ID',
  `title`       VARCHAR(100) NOT NULL COMMENT '事件标题',
  `start_time`  DATETIME     NOT NULL COMMENT '开始时间',
  `end_time`    DATETIME     NOT NULL COMMENT '结束时间',
  `all_day`     TINYINT(1)   NOT NULL DEFAULT 0 COMMENT '是否全天 1是0否',
  `location`    VARCHAR(200) DEFAULT NULL COMMENT '地点',
  `note`        VARCHAR(500) DEFAULT NULL COMMENT '备注',
  `recurrence`  VARCHAR(20)  NOT NULL DEFAULT 'none' COMMENT '重复规则 none/daily/weekly/monthly',
  `status`      TINYINT(1)   NOT NULL DEFAULT 1 COMMENT '状态 1有效0已删除',
  `create_time` DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  `update_time` DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  PRIMARY KEY (`id`),
  KEY `idx_user_start` (`user_id`, `start_time`),
  KEY `idx_user_status` (`user_id`, `status`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='事件表';
```
> 索引说明：`idx_user_start` 服务于今日/周/月视图与冲突检测的「按用户 + 时间范围」查询；`idx_user_status` 服务于逻辑删除过滤。

### 3.3 reminder（提醒表）
对应功能：F-REMIND-01 到点提醒、F-REMIND-02 自定义提前量。

| 字段 | 类型 | 是否必填 | 默认值 | 说明 |
| --- | --- | --- | --- | --- |
| id | BIGINT UNSIGNED | 是 | AUTO_INCREMENT | 主键，提醒 ID |
| event_id | BIGINT UNSIGNED | 是 | — | 关联事件（FK → event.id） |
| user_id | BIGINT UNSIGNED | 是 | — | 冗余用户 ID，便于按用户扫描 |
| offset_minutes | INT | 是 | 15 | 提前分钟数（F-REMIND-02） |
| fire_time | DATETIME | 是 | — | 实际触发时间 = start_time − offset_minutes |
| sent | TINYINT(1) | 是 | 0 | 是否已推送，1 是 0 否 |
| create_time | DATETIME | 是 | CURRENT_TIMESTAMP | 创建时间 |

```sql
CREATE TABLE `reminder` (
  `id`             BIGINT UNSIGNED NOT NULL AUTO_INCREMENT COMMENT '提醒ID',
  `event_id`       BIGINT UNSIGNED NOT NULL COMMENT '关联事件ID',
  `user_id`        BIGINT UNSIGNED NOT NULL COMMENT '用户ID(冗余)',
  `offset_minutes` INT          NOT NULL DEFAULT 15 COMMENT '提前分钟数',
  `fire_time`      DATETIME     NOT NULL COMMENT '触发时间',
  `sent`           TINYINT(1)   NOT NULL DEFAULT 0 COMMENT '是否已推送 1是0否',
  `create_time`    DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  PRIMARY KEY (`id`),
  KEY `idx_event` (`event_id`),
  KEY `idx_fire_sent` (`fire_time`, `sent`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='提醒表';
```
> `idx_fire_sent` 服务于调度器每分钟扫描 `WHERE fire_time <= now AND sent = 0`。

### 3.4 voice_log（语音解析日志表）
对应功能：F-VOICE-01/02/03 解析、F-SET-02 隐私清除。

| 字段 | 类型 | 是否必填 | 默认值 | 说明 |
| --- | --- | --- | --- | --- |
| id | BIGINT UNSIGNED | 是 | AUTO_INCREMENT | 主键 |
| user_id | BIGINT UNSIGNED | 是 | — | 用户 ID |
| raw_text | VARCHAR(500) | 是 | — | 语音转写/输入的原文 |
| intent | VARCHAR(20) | 否 | NULL | 解析意图：create/query/update/delete/unknown |
| parsed_json | JSON | 否 | NULL | 大模型返回的结构化 JSON |
| create_time | DATETIME | 是 | CURRENT_TIMESTAMP | 创建时间 |

```sql
CREATE TABLE `voice_log` (
  `id`          BIGINT UNSIGNED NOT NULL AUTO_INCREMENT COMMENT '主键',
  `user_id`     BIGINT UNSIGNED NOT NULL COMMENT '用户ID',
  `raw_text`    VARCHAR(500) NOT NULL COMMENT '识别/输入原文',
  `intent`      VARCHAR(20)  DEFAULT NULL COMMENT '解析意图',
  `parsed_json` JSON         DEFAULT NULL COMMENT '结构化解析结果',
  `create_time` DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  PRIMARY KEY (`id`),
  KEY `idx_user_time` (`user_id`, `create_time`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='语音解析日志表';
```

## 5. 关键 SQL（与功能对应）

### 5.1 冲突检测（F-CONFLICT-01）—— 创建/修改前调用
```sql
SELECT id, title, start_time, end_time
FROM `event`
WHERE user_id = #{userId}
  AND status = 1
  AND start_time < #{newEndTime}
  AND end_time   > #{newStartTime}
  AND id <> #{selfId};   -- 修改时排除自身
```

### 5.2 今日 / 周 / 月视图与查询（F-VIEW-01/02/03）
```sql
SELECT * FROM `event`
WHERE user_id = #{userId} AND status = 1
  AND start_time < #{rangeEnd}
  AND end_time   > #{rangeStart}
ORDER BY start_time ASC;
```

### 5.3 搜索（F-VIEW-05，分页）
```sql
SELECT * FROM `event`
WHERE user_id = #{userId} AND status = 1
  AND (title LIKE CONCAT('%', #{keyword}, '%')
       OR location LIKE CONCAT('%', #{keyword}, '%'))
ORDER BY start_time DESC
LIMIT #{offset}, #{pageSize};
```

### 5.4 逻辑删除与撤销（F-DEL-01/02）
```sql
-- 删除
UPDATE `event` SET status = 0 WHERE id = #{id} AND user_id = #{userId};
-- 撤销（5 秒内）
UPDATE `event` SET status = 1 WHERE id = #{id} AND user_id = #{userId};
```

### 5.5 提醒调度扫描（F-REMIND-01）
```sql
SELECT r.*, e.title, e.start_time
FROM `reminder` r JOIN `event` e ON r.event_id = e.id
WHERE r.sent = 0 AND r.fire_time <= NOW() AND e.status = 1;
-- 推送后
UPDATE `reminder` SET sent = 1 WHERE id = #{id};
```

### 5.6 清除语音数据（F-SET-02）
```sql
DELETE FROM `voice_log` WHERE user_id = #{userId};
```

## 7. 设计要点说明
1. **逻辑删除**：`event.status` 不做物理删除，满足「删除可 5 秒撤销」(F-DEL-02)，所有查询都带 `status = 1`。
2. **提醒独立成表**：一个事件可挂多条提醒（F-REMIND-02），修改事件时间需级联重算 `reminder.fire_time`。
3. **设置内嵌 user 表**：一期偏好项少，直接做成 `user` 列，避免额外联表；后续可拆 `user_setting` 表。
4. **voice_log 与隐私**：受 `user.voice_retention` 控制，由定时任务按策略清理，用户也可手动清除（F-SET-02）。
5. **时间一致性**：服务端以用户 `timezone` 解析相对时间并写库，避免客户端时间不准。
