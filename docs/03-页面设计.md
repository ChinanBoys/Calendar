# 语音日历 VoiceCal —— 页面设计说明

> 配套 `01-需求文档(PRD).md`。本文档描述每个页面的布局、控件及其对应的需求编号，并与可点击原型 `prototype/` 一一对应。
> 设计原则：**移动端优先**；**语音按钮全局可达**；**先确认后保存**；**破坏性操作可撤销**。

设计风格：圆角卡片、留白充足、主色 `#5B6CFF`（靛蓝）+ 强调色 `#FF7A59`（珊瑚橙，用于语音激活态），冲突告警用红 `#E5484D`，低置信度高亮用黄 `#F5A623`。

---

## P1 首页 / 今日视图（核心页）

### 布局（自上而下）
```
┌─────────────────────────────────────┐
│ ☰  2026年6月1日 周一        🔍   🔔   │  顶栏
├─────────────────────────────────────┤
│ [ 今日 ] [ 周 ] [ 月 ]               │  视图切换 Tab
├─────────────────────────────────────┤
│  今日时间轴                          │
│  ─ 09:00  晨会          3号会议室 >   │  ← 事件卡（点击进详情，左滑删除）
│  ─ 15:00  会议(冲突⚠)               │
│  ─ 19:00  健身                       │
│  （空态：今天还没有安排，说一句试试） │
├─────────────────────────────────────┤
│                                      │
│            ●●●  波纹动画              │
│          ┌───────────┐               │
│          │    🎤      │  ← 中央语音按钮(长按说话) │
│          └───────────┘               │
│       "按住说话，如：明天3点开会"      │
│                                      │
│   [ ⌨ 输入 ]            [ 我有什么安排 ]│  文本兜底 / 语音查询快捷
└─────────────────────────────────────┘
```

### 控件 → 需求映射
| 控件 | 交互 | 对应需求 |
| --- | --- | --- |
| 顶栏 `☰` 菜单 | 打开设置 | 跳 P7 / F-SET |
| 顶栏 `🔍` 搜索 | 进入搜索页 | 跳 P6 / F-VIEW-05 |
| 顶栏 `🔔` 铃铛 | 进入通知中心 | 跳 P5 / F-REMIND-01 |
| 视图 Tab `今日/周/月` | 切换视图 | F-VIEW-01 / F-VIEW-02 |
| 时间轴事件卡 - 点击 | 进入事件详情 | 跳 P4 / F-VIEW-04 |
| 时间轴事件卡 - 左滑「删除」 | 删除并提供撤销 | F-DEL-02 |
| 事件卡上的 ⚠ 标记 | 标识时间冲突 | F-CONFLICT-01 |
| **中央语音按钮 🎤** | 长按/单击录音 → 转写 → 解析 | F-VOICE-01 → F-ADD-01 |
| `⌨ 输入` 按钮 | 弹出文本框，输入自然语言 | F-VOICE-02 |
| `我有什么安排` 快捷 | 直接触发语音查询 | F-VIEW-03 |

### 语音按钮状态
- 默认：靛蓝静止。
- 录音中：珊瑚橙 + 波纹动画 + 实时显示识别文字。
- 解析中：转圈 loading。
- 解析完成：弹出 P2 确认卡。

## P2 解析结果确认卡（先确认后保存的关键体验）

从 P1 语音/文本解析完成后由底部弹出。

### 布局
```
┌─────────────────────────────────────┐
│ 确认这条日程                      ✕  │
├─────────────────────────────────────┤
│ ⚠ 与「周会 14:30-15:30」时间冲突      │  ← 仅冲突时出现(红)
├─────────────────────────────────────┤
│ 原话：我明天下午三点到五点有个会议    │
│                                      │
│ 标题     会议                        │
│ 日期     6月1日 周一                  │
│ 开始     15:00                       │
│ 结束     17:00                       │
│ 地点     3号会议室   (黄=需核对)      │  ← 低置信度字段高亮
│ 提醒     提前15分钟                   │
│ 重复     不重复                       │
├─────────────────────────────────────┤
│ [ 🎤 重说 ]  [ 手动修改 ]            │
│ [        确认保存        ]           │
│ [ 取消 ]                             │
└─────────────────────────────────────┘
```

### 控件 → 需求映射
| 控件 | 交互 | 对应需求 |
| --- | --- | --- |
| 字段展示（标题/日期/起止/地点/提醒/重复） | 渲染解析 JSON，低置信度字段黄色高亮 | F-VOICE-03 |
| 红色冲突提示条 | 冲突告警，可【仍然保存】 | F-CONFLICT-01 |
| `🎤 重说` | 二次语音修正字段 | F-VOICE-04 |
| `手动修改` | 进入编辑表单（P4） | F-EDIT-02 |
| `确认保存` | `POST /api/events` | F-ADD-01 |
| `取消` / `✕` | 放弃 | — |

> 不同 intent 下确认卡形态：`create` 显示完整字段；`delete` 显示「将删除：xx」红色二次确认（F-DEL-01）；`update` 显示「将把 xx 从 A 改为 B」（F-EDIT-01）。

## 全局交互规范
| 规范 | 说明 | 关联需求 |
| --- | --- | --- |
| 语音按钮全局可达 | 首页中央 + 其余页悬浮 | F-VOICE-01 |
| 先确认后保存 | 任何由语音产生的增删改都先过确认卡 | F-VOICE-03 |
| 删除可撤销 | 删除后底部 toast「已删除 · 撤销」5 秒 | F-DEL-02 |
| 低置信度高亮 | 黄色提示用户核对 | F-VOICE-03 |
| 冲突红色告警 | 顶部红条 + 事件卡 ⚠ | F-CONFLICT-01 |
| 文本兜底 | 任何语音入口旁都有文本输入 | F-VOICE-02 |
